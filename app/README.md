# FlowMap.app — macOS desktop shell

A double-clickable macOS app that bundles the FlowMap WebGL2 client **and** the
FlowMap Python server, so there is nothing for the user to install — no Python,
no Node, no `uv`. It is a [Tauri 2](https://tauri.app) shell (Rust + native
WKWebView) that:

1. picks a free loopback TCP port;
2. spawns a **bundled, relocatable Python 3.13** running `flowmap_server` on that
   port as a sidecar (recording under the app-data dir);
3. injects `window.__FLOWMAP_SERVER__ = "http://127.0.0.1:<port>"` into the
   webview before the client loads, so the SPA reaches the sidecar;
4. terminates the sidecar on quit (SIGTERM → SIGKILL) — no orphan process.

## Layout

```
app/
  icons/                     hand-authored source icon (flowmap.svg, .icns, PNGs)
  scripts/
    bundle-python.sh         builds the relocatable pyruntime (astral python-build-standalone)
    build-dmg.sh             full pipeline: client → pyruntime → tauri build → sign → dmg
  src-tauri/
    tauri.conf.json          product/identifier/version + bundle config
    Cargo.toml               Rust deps (tauri 2, libc)
    build.rs
    src/main.rs              sidecar lifecycle (spawn / health / teardown)
    capabilities/default.json
    icons/                   generated by `cargo tauri icon` (gitignored)
    resources/pyruntime/     the bundled Python (gitignored — ~529 MB)
```

The bundled `pyruntime/`, the Rust `target/`, the generated `icons/`, and the
built `*.app`/`*.dmg` are **gitignored** (large binaries). The DMG ships via the
GitHub Release, not git.

## Build

```bash
# from the repo root — one command does everything:
bash app/scripts/build-dmg.sh
```

Outputs:

- `app/src-tauri/target/release/bundle/macos/FlowMap.app` (~536 MB)
- `app/src-tauri/target/release/bundle/dmg/FlowMap_1.0.0_aarch64.dmg` (~169 MB)

Requires: `cargo`/`rustc`, `cargo-tauri` (`cargo install tauri-cli --version '^2'
--locked`), Node/npm, `uv`, and the macOS `codesign`/`hdiutil`/`sips`/`iconutil`
tools.

## First run (unsigned / unnotarized)

Only an **Apple Development** signing identity is available (not a paid Developer
ID), so the app + DMG are **ad-hoc signed** — Gatekeeper will warn on first
launch. To open it:

- **Right-click FlowMap.app → Open** (then confirm the dialog), or
- `xattr -cr /Applications/FlowMap.app` to clear the quarantine attribute.

This is a consequence of the missing Developer ID, not a bug. Notarization is out
of scope.

## Run from source (alternative)

If you prefer not to use the DMG (or aren't on Apple Silicon macOS), run from
source — see the repo root `README.md` and `scripts/dev.sh`.
