### Repro
1. Use REST path with hung network / blackhole route.
2. `_poll_tick` calls `fetch_order_book` without configured short timeout.
3. GUI remains blocked until TCP stack timeout (tens of seconds to minutes).

### Expected
`timeout` (ms) in `_ccxt_config` (e.g. 5000–10000) + cancel on disconnect.

### Actual
```python
self._ccxt_config: dict[str, Any] = {
    "enableRateLimit": True,
    "options": {"defaultType": "spot"},
}
# no "timeout" key
```

### Fix hint
Add `"timeout": 10000` (or similar); never run hang-prone fetch on GUI thread (FIND-P223-01).
