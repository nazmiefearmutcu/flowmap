### Repro
1. Drag history left → `auto_follow=False`, `_scroll_offset>0`, "Go Live" visible.
2. Press **F** (MainWindow): `set_auto_follow(not auto_follow)` → status "Auto-follow: ON".
3. Until next `push_snapshot`, view still shows history with offset; if toggled OFF while at offset 0, no rebuild either.
4. Contrast **L/Esc** / Go Live click: sets `auto_follow=True`, `_scroll_offset=0`, `rebuild_heatmap()`.

### Expected
Enabling follow from history is equivalent to Go Live (offset 0 + rebuild). Disabling follow freezes current live edge intentionally.

### Actual
Status says ON while scrolled; next push forces offset=0 only when already True at push time — interim desync and asymmetric F vs L.

```1034:1035:flowmap/ui/heatmap_widget.py
    def set_auto_follow(self, e: bool) -> None:
        self.auto_follow = e
```

### Fix hint
`set_auto_follow(True)` → `_scroll_offset=0; rebuild_heatmap()`; False leaves offset. Or F only when not scrolled and use Go Live for return-to-live.
