### Repro
1. After equal-time bin loop, code computes `gap_ns = time.time_ns() - last_ts` with `last_ts = end_ns` (window end from lake max trade or wall clock).
2. `gap_bins = int(gap_ns / bin_duration)`; if `gap_bins >= bw` → reset order_book, pulse, VP, **heatmap** (discard all bin work).
3. Stale lake: max trade days/weeks before “now” → gap_bins ≫ bw → wipe always.
4. With FIND-P236-01 (`bw=1`), any gap ≥ one bin duration triggers wipe.

### Expected
Successful hist remains visible ending at last data (or explicit user message before clear). Wall-clock gap must not silently erase preload.

### Actual
Stale or offline lakes pay full hist cost then show blank canvas. No toast/status. Hold-forward branch freezes last book into “present” columns when gap &lt; bw (also misleading).

### Fix hint
Default: never wipe successful hist; cap hold-forward at `bw-1` or anchor timeline to `end_ns`. If wipe retained: status “Data is N hours old; clearing canvas”. Config `gap_policy=hold|wipe|none`. Tests with frozen clock.
