### Repro
1. First live `start()` patches `aiohttp.ClientSession.ws_connect` and sets `_patched_for_flowmap = True`.
2. Wrapper always does `kwargs['ssl'] = False` — overrides any explicit SSLContext.
3. `disconnect()` never restores original method; original not stored for undo.
4. Entire process (including embed host) remains patched for all subsequent `ws_connect` calls.

### Expected
Scoped transport SSL config; reversible; default verify; insecure only via explicit env/UI warning.

### Actual
Process-global, permanent after first live start; silent MitM surface (primary issue FIND-P218-01). This FIND adds irreversibility + force-override blast radius.

### Fix hint
Remove class monkeypatch; pass ssl into transport/session factory local to worker; gate insecure with `FLOWMAP_WS_INSECURE=1`.

### Evidence
```100:108:flowmap/data/crypcodile_live.py
async def patched_ws_connect(...):
    kwargs['ssl'] = False
    return await original_ws_connect(...)
aiohttp.ClientSession.ws_connect = patched_ws_connect
aiohttp.ClientSession._patched_for_flowmap = True
```
No restore on disconnect.
