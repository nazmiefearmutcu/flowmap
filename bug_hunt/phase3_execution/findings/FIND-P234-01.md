### Repro
1. Heatmap: fixed pitch `y = i * row_height` via `_price_to_screen_y` / vis_rows = h // row_height (remainder strip unused).
2. VP: `y_start = int(i * h / bh)` distributing full widget height across `bh` levels.
3. Set e.g. h=405, row_height=4 → heatmap vis_rows=101; VP stretches all rows over 405 px → cumulative skew grows with i.
4. `set_row_height` stores `self.row_height` but paint never uses it.

### Expected
VP row i Y matches heatmap row i (shared helper or `i * row_height`), including unused bottom remainder.

### Actual
Systematic price-bar misalignment; traders cannot trust COB/CVP/SVP vs heatmap strata.

```328:333:flowmap/ui/overlays/volume_profile.py
        bh = len(self._levels)
        for i, level in enumerate(self._levels):
            price = level.price
            y_start = int(i * h / bh)
            y_end = int((i + 1) * h / bh)
```

### Fix hint
`y_start = i * self.row_height` (clamped); sync levels from `get_visible_prices()` only; drop stretch-to-fill. Quant pass: max |y_vp - y_hm| < 1 px for row_height ∈ {2,4,8,12}.
