### Repro
1. Code always does `sys.path.insert(0, "/Users/nazmi/flowmap")` if not already present.
2. Even when `pip install -e` / site-packages provides `flowmap`, insert(0) forces developer tree **first** when that path exists.
3. Wrong commit / dirty tree loads silently — `flowmap.__file__` points at hardcoded path, not installed pin.
4. No `FLOWMAP_HOME` / version pin / Crypcodile packaging dependency on flowmap (grep pyproject/setup: no flowmap dep).

### Expected
Prefer installed package; optional env override; clear error if missing. Declare dependency + version pin.

### Actual
Hardcoded path wins when present; no pin; dual-tree drift with P2-42. Related FIND-P235-01 is the missing-path ImportError; this finding is **shadowing when path exists**.

### Fix hint
Remove inject; `pip install -e ../flowmap`. Interim: `FLOWMAP_HOME` only if set; never insert over existing import without debug flag.

### Evidence
```python
flowmap_path = "/Users/nazmi/flowmap"
if flowmap_path not in sys.path:
    sys.path.insert(0, flowmap_path)
```
Crypcodile packaging metadata has no `flowmap` dependency.
