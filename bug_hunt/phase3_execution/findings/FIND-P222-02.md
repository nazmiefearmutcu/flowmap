### Repro
1. `self._queue = queue.Queue()` only in `__init__` — never replaced on switch/stop.
2. Producers put bare `("snapshot", obj)` with **no** generation/session id.
3. Race: `stop_current` drain returns → orphan/dying worker `put` → new session `Start` applies foreign symbol/source data.
4. TOCTOU: between last `get_nowait` and drain exit, a put lands; residual survives into next session.

### Expected
Epoch token per start/switch; consumer drops mismatched session; optional new Queue per session so dying workers write to abandoned queue.

### Actual
Shared unstamped queue for app lifetime; late puts indistinguishable from new session traffic.

### Fix hint
`self._session_id += 1` on start; tuple `(session_id, type, obj)`; drop mismatch in `_gui_tick`. On switch, optionally replace `_queue` after stop.

### Evidence
- Queue created once `:82`
- Drain loop no epoch `:194-200`
- Put format 2-tuple in live/replay/crypto
