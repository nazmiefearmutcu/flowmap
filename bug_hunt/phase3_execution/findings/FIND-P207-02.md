### Repro (unit probe)

```python
import numpy as np
from flowmap.engine import DensityEngine
from flowmap.core import BookLevel, BBO

e = DensityEngine(history_width=16, max_levels=100, decay=0.92)
e.tick_size = 0.1
e._tick_size_detected = True
e.ticks_per_row = 10          # render_tick = 1.0 → many ticks → one row
e.vertical_smoothing = 0.0
e.min_order_size = 0.0
e.resize(40, 16)
# Two bid levels that map to the SAME render row (prices 100.1 and 100.5 both round into same bin near 100)
levels = [
    BookLevel(price=100.1, bid_size=10.0),
    BookLevel(price=100.5, bid_size=100.0),
    BookLevel(price=101.0, ask_size=10.0),
]
bbo = BBO(0, "T", 99.9, 100.0, 1, 1)
e.center_price_ticks = int(round(100.0 / e.render_tick_size))
e._center_price_ticks_float = float(e.center_price_ticks)
# Direct aggregation check mirrors engine:
buf_h = e._buffer.shape[0]
bid_arr = np.zeros(buf_h)
prices = np.array([100.1, 100.5])
vals = np.array([10.0, 100.0])
rows = (buf_h//2) - np.round(prices / e.render_tick_size).astype(np.int32) + e.center_price_ticks
np.maximum.at(bid_arr, rows, vals)
assert bid_arr[rows[0]] == 100.0   # max, not 110
# sum would be 110 — understates stacked ladder liquidity by 10
```

### Expected
When `ticks_per_row > 1`, multiple exchange ticks collapse into one visual row. Stacked liquidity should be **summed** (Bookmap-style wall thickness) or at least documented.

BTC default `ticks_per_row=100` collapses ~100 ticks → 1 row; only the **largest single level** survives. Thin multi-level walls understate total resting size → weaker color / invisible structure.

### Unit result
| Probe | tpr | sizes | Aggregated | Sum |

### Fix hint
Replace `np.maximum.at` with `np.add.at` (guard overflow). Optionally keep max as config `row_agg: max|sum`. Critical for BTC path (tpr=100).
