2. Grep: no `paintGL` / `initializeGL` — both bases share `paintEvent` + QPainter + QImage.
3. Print claims "GPU-accelerated QOpenGLWidget" despite CPU painter path.
4. CI/scripts hit CPU path; packaged default hits QOpenGLWidget (`render`/`grabFramebuffer` fallback).

### Expected
Honest backend naming; dual-backend smoke tests; safe fallback if GL context fails at show.

### Actual
Surface-only difference; production GL compositing/DPR/`render()` quirks under-tested; misleading GPU log.

### Fix hint
Rename log to "QOpenGLWidget surface"; add offscreen dual-backend pixel parity test; ensure ImportError and context-fail fall back to QWidget without crash.
