### Repro
1. **Full path** `stop_current`: disconnect provider signals → stop_replay → disconnect → `provider=None` → **drain queue** → `running=False`.
2. **User Stop** `_toggle_live` (`:497-499`): only `provider.disconnect()` + `running=False` — **no** `_disconnect_provider_signals`, **no** drain, provider object retained.
3. **User Stop** `_toggle_replay` (`:455-459`): stop_replay + disconnect + running=False — same gaps.
4. Provider disconnect may orphan threads (FIND-P217/219); signals remain connected on live provider object.
5. Rapid switch with hung worker: new provider starts while old thread still puts.

### Expected
Unified teardown: disconnect all signals (worker+provider), join or retain ownership, drain, single worker invariant.

### Actual
Two stop quality levels; UI Stop is the incomplete one. `closeEvent` uses `stop_current` (better) but still inherits wait-timeout orphans.

### Fix hint
`teardown_provider()` used by toggles, switch_to, symbol change, close; refuse start while previous thread running.

| `stop_current` | yes | yes | yes |
| `_toggle_live` stop | no | no | no |
| `_toggle_replay` stop | no | no | no |
