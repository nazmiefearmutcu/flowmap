### Repro
1. `_gui_tick` order: `heatmap.add_trades(trades)` then `heatmap.push_snapshot(...)`.
2. `add_trade` stores `tick_index = self._frame_count` (N).
3. `push_snapshot` does `_frame_count += 1` → N+1 for that live column.
4. Draw: `col = bw - 1 - frame_count + tick_index` → with frame=N+1, tick=N → **bw-2** (one left of live edge bw-1).

### Expected
Co-batched trades share the same column index as the snapshot written in the same GUI tick.

### Actual
Deterministic 1-column lag for dots and bubbles at the live wall.

### Fix hint
Stamp trades with `self._frame_count + 1` when trades are applied before push in same tick, or increment frame before trades, or pass explicit column index from tick orchestrator.
