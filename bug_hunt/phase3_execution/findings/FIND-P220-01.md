### Repro
1. Production always constructs providers with `queue=self._queue` (`source_manager.py:256`, `:301`).
2. Workers: `if self._queue is not None: put else: sig_*.emit` — market data signals never fire in prod.
3. SourceManager **still** connects `on_snapshot/update/trade/bbo` → `_on_provider_*` which call `apply_snapshot` / `record_trade` directly (`:328-339`).
4. Worker→provider `sig_*` relays also wired (`crypcodile_live.py:239-242`) but unused for data.
5. If an engineer removes the `if queue` branch or sets `queue=None` while keeps both connects + `_gui_tick` drain → **double book apply**.

### Expected
Single market-data path documented and enforced (XOR assert in debug).

### Actual
Dead dual wiring: lifecycle signals live; data signal handlers dead in prod; latent P0 if exclusivity broken. `_on_provider_bbo` is `pass` even if signals revived.

### Fix hint
When queue mode: only connect lifecycle/error/progress. Drop or guard `_on_provider_*` data applies. Optional runtime assert: queue mode ⇒ no data signal emits.
