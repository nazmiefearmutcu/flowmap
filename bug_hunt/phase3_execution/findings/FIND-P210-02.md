### Geometry

Buffer height `bh = vis_rows * 5` (overscan). Paint blits **center slice only**:

```1177:1183:flowmap/ui/heatmap_widget.py
                        vis_rows = max(1, cache_h // self.row_height)
                        start_row = bh // 2 - vis_rows // 2
                        cache_painter.drawImage(
                            QRect(hm_left, 0, timeline_w, cache_h),
                            qimg,
                            QRect(0, start_row, bw, vis_rows)
                        )
```

History helper maps buffer row index through **full** height:
```python
y_scale = self.height() / bh          # = row_height / 5 when bh=5*vis
rows = half_bh - (p_ticks - center)
ys = rows * y_scale + y_offset
```

`_price_to_screen_y` maps visible rows:
```python
screen_row = (vis_rows // 2) - (p_ticks - center)
y = screen_row * row_height + row_height / 2
```

### Unit equivalence (tpr=1, correct tick — isolates scale bug)

| Δ render ticks from center | y_hist (full buffer) | y_vis (`_price_to_screen_y`) | Ratio of displacement |
|----------------------------|----------------------|------------------------------|------------------------|
| 0 | ~height/2 | ~height/2 | — |
| 1 | moves by `height/bh` ≈ `rh/5` | moves by `row_height` | **5× compressed** |
| 10 | 10·rh/5 = 2 rh | 10·rh | **5×** |

Overscan factor 5 is hard-coded in `resize` (`buf_h = vis_rows * 5`). History polylines compress price motion into 1/5 of correct screen travel even after FIND-P210-01 tick fix.

Live BBO badges (paint overlay via `_price_to_screen_y`) end of mid line (history) **diverge** at same price.

### Expected
History polyline Y must use same transform as heatmap blit (visible slice + row_height) and `_price_to_screen_y`.

### Actual
Independent full-buffer linear map → systematic vertical compression × overscan ratio (5).

### Fix hint
Replace helper body with vectorized `_price_to_screen_y` (pass `wh=self.height()`). Do not use `height/bh` unless drawing in raw buffer pixel space before the same center-slice crop.
