### Repro
1. Set `order_book.on_trade = wrapper` that chains to MainWindow and plugin.
2. Run successful `_gui_tick` with trades.
3. After restore, `on_trade is self._on_trade` — wrapper gone.

### Expected
Restore previous callable (`prev = on_trade` / finally assign `prev`).

### Actual
Hard assignment to `self._on_trade` destroys any external wrapper every tick that has market data.

### Fix hint
Save/restore previous handler; or `notify=False` flag so attribute is never swapped. Define plugin chain API before P2-46 enablement.

### Evidence
Line `:947` assigns `self._on_trade` unconditionally, not prior value.
