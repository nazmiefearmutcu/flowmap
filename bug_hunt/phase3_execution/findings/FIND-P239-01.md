### Repro
1. Open a lake where `trade` spans multi-day while selected UI range is a short book window (`start_ns`/`end_ns`).
2. Start Crypcodile replay for that symbol/range.
3. Inspect emitted `Trade.timestamp` vs original `Record.local_ts` (or log `MappedRecord.local_ts`).
4. Observe trades from outside the book window appear linearly remapped into `[start_ns, end_ns]`.

### Expected
Trades loaded with the same `frm`/`to` as books; timestamps preserve market causality with the book series.

### Actual
Books: `replay(..., frm=start_ns, to=end_ns)`.  
Trades: `MIN/MAX(local_ts)` over **entire** trade table, then:

```text
scale = book_span / t_span
mapped_ts = start_ns + (trade_ts - t_min_actual) * scale
```

Every trade (and liquidation — same channel batch) is stretched/compressed into the book window. Progress uses mapped timestamps so the UI looks “correct” while data is fictional.

### Fix hint
Use identical `frm`/`to` for trade/liquidation as books. Drop global remap, or gate behind explicit “demo align” flag with UI disclosure. Stream-merge by real `local_ts` without full-table MIN/MAX.
