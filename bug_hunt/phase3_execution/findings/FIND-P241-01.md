### Repro
1. Set symbol to `BTC' OR '1'='1` (or any string containing `'`).
2. Start replay or call `get_time_range` / `load_symbols`.
3. Query fails or returns unintended rows; exceptions often swallowed → empty range / spin path.

### Expected
Parameterized DuckDB queries (`?` / `$1`) or strict symbol allowlist `[A-Za-z0-9:._\-/]+`.

### Actual
UI free-text symbol interpolated via f-strings. Local DuckDB limits blast radius but correctness/DoS/injection within lake remain real; MotherDuck/remote future elevates risk.

### Fix hint
```python
client.query("SELECT MIN(local_ts), MAX(local_ts) FROM trade WHERE symbol = ?", [symbol])
```
Validate symbol before any SQL; never interpolate `table`/`channel` from user input (currently enum-ish — keep hard allowlist).
