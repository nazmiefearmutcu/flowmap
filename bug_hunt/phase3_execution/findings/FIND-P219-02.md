### Repro
1. Start replay with wall-clock fallback range that has no lake rows (`source_manager` fallback last 1h).
2. `records = list(book_iter)` → empty; emit for-loop no-ops.
3. End of pass: `if self._running: print(... auto-looping...);` then outer `while self._running` immediately reloads.
4. Tight loop of DuckDB/client.replay with no backoff while Start remains active.

### Expected
Empty pass → stop with clear error or long backoff; no busy loop.

### Actual
Auto-restart without sleep when empty → CPU spin + repeated SQL until user Stop (stop still blocked if mid-query).

### Fix hint
If `len(records)==0`: emit error, set `_running=False`, break; optional user “loop replay” toggle only when data existed.

### Evidence
```539:541:flowmap/data/crypcodile_replay.py
if self._running:
    print("[REPLAY_WORKER] Replay finished, auto-looping/restarting from beginning...")
```
No empty-data guard before outer while restart.
