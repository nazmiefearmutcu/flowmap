### Repro
1. Spec fields:
   - `datas=[]`, `hiddenimports=[]`, `excludes=[]`
   - `upx=True` on EXE **and** COLLECT
   - BUNDLE: `icon=None`, `bundle_identifier=None`
2. Built `Info.plist`: `CFBundleIdentifier=FlowMap`, `CFBundleShortVersionString=0.0.0` (not setup.py `0.1.0`).
3. Dist pulls heavy transitive tree (web3/eth/pyarrow/numba/…) via analysis — cold start bloat, more dylib failure surface.
4. UPX on macOS historically breaks some signed/dylib loads.

### Expected
`upx=False` on Darwin release; explicit hiddenimports for lazy paths if needed; reverse-domain bundle id; version synced from setup.py; aggressive excludes for unused eth/web3 if safe.

### Actual
Minimal bare spec; product version 0.0.0; UPX enabled; no packaging hygiene gates.

### Fix hint
Release checklist from P2-48 §3.1; CI fail on `bundle_identifier is None` / version 0.0.0; rebuild matrix upx on/off.
