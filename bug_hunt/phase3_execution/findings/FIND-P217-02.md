### Repro
1. Worker `start()` sets event loop and blocks on `run_until_complete(self._run())` — no Qt event loop pumping on that thread for `quit()`.
2. `disconnect` calls `thread.quit()` — no-op until `_run` returns.
3. `stop()` only sets `_running=False` and, if transport exists, schedules `transport.close()`; no task cancel, no `loop.stop` if transport is None.
4. Race: stop before connector assigned → only flag; run may still block in `make_connector` / await.

### Expected
Deterministic cooperative cancel of `connector.run` and loop exit; `quit` not relied on for asyncio workers.

### Actual
Teardown depends entirely on transport.close unblocking await; quit is cargo-cult. Orphan path (FIND-P217-01) is structural.

### Fix hint
Threadsafe shutdown coroutine: cancel tasks, close transport, stop loop. Document that asyncio workers must not use quit-as-primary.

### Evidence
```112:133:flowmap/data/crypcodile_live.py
self._loop.run_until_complete(self._run())

def stop(self) -> None:
    self._running = False
    if self._loop and self._loop.is_running():
        if self._connector and self._connector.transport:
            asyncio.run_coroutine_threadsafe(
```
