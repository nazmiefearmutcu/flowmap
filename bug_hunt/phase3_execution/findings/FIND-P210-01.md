### Repro (numeric Y matrix)

Correct grid (`_price_to_screen_y`):
```python
p_ticks = round(price / engine.render_tick_size)  # tick * tpr
```

History line (bug):
```1408:1418:flowmap/ui/heatmap_widget.py
        tick_size = self._engine.tick_size
        ...
        p_ticks = np.round(prices_arr / tick_size)
```

| Probe | price | tick | tpr | render | p_ticks raw | p_ticks render | Equal? |
|-------|-------|------|-----|--------|-------------|----------------|--------|
| Y1 | 100.0 | 0.1 | 1 | 0.1 | 1000 | 1000 | yes |
| Y2 | 100.0 | 0.1 | 10 | 1.0 | 1000 | 100 | **no** |
| Y3 | 100.0 | 0.1 | 100 | 10 | 1000 | 10 | **no** (BTC default) |
| Y4 | mid series | | 100 | | max \|Δ ticks\| = 990× | | systematic |

Density / live BBO buffer stamp / trades / axis all use `render_tick_size`. History mid/bid/ask polylines alone use raw tick → when `ticks_per_row≠1` (always BTC path), line Y is ~`tpr`× wrong vs heatmap structure.

### Expected
For tpr ∈ {1,2,10,100}, history line Y for price P equals `_price_to_screen_y(P)` within 1 px (after also fixing FIND-P210-02 scale).

### Actual
`tick_size = self._engine.tick_size` then `prices / tick_size` while `center_price_ticks` is in **render** tick units → vertical mis-scale factor ≈ `ticks_per_row`.

### Fix hint
```python
tick_size = self._engine.render_tick_size
```
Prefer reusing `_price_to_screen_y` (or vectorized twin) so tick + visible-row mapping stay single-source.
