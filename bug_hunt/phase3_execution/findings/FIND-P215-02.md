### Repro
1. Same tick batch: update U, trade T, snapshot S (FIFO collection).
2. Snapshot clears U from `updates` list ‚Äî good for L2 replace.
3. Apply order is always: last snapshot ‚Üí remaining updates ‚Üí last bbo ‚Üí **all** trades.
4. Any trade that occurred before S in the queue still runs after S (FIND-P215-01).
5. Multi-tick with cap 1000: first tick applies 1000 updates; second tick may apply snapshot ‚Äî intermediate wrong book is painted via `push_snapshot` each non-empty tick.

### Expected
Market events applied in stream order (or equivalent semantics for full snapshots).

### Actual
Type-batched reordering within a tick; multi-tick partial drains expose intermediate states to heatmap.

### Fix hint
Chronological apply in drain order with budget; only last heatmap push per tick if needed for perf.

### Evidence
Apply block order fixed at `:937-944`; `push_snapshot` every non-empty tick at `:959`.
