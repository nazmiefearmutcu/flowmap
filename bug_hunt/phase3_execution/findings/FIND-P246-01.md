### Repro
1. Grep production call sites: `load_all_from_directory|discover_plugins|load_and_register` — hits only under `flowmap/plugins/*` and `plugins/example_*` (not `main.py` / `MainWindow`).
2. `load_plugin` uses `importlib.util.spec_from_file_location` + `spec.loader.exec_module(module)` with no sandbox, hash allowlist, or timeout.
3. Hostile plugin with `register(api)` writing a file / spawning subprocess would run as the FlowMap user.
4. Docs in `plugins/example_indicator.py` claim “auto-loaded at startup” — false today, dangerous if taken as product intent.

### Expected
Security model B (opt-in, allowlisted path, hash/signature or subprocess host) **before** any MainWindow wire. Default disabled. Docs match reality.

### Actual
Full RCE surface implemented and ready; currently **unwired** → latent P0. Naive wire + cwd `plugins/` = ship-blocker. Packaging must not auto-scan untrusted dirs.

### Fix hint
Model A short-term: keep disabled, fix docs. Model B: Application Support plugins dir + SHA256 allowlist + read-only OrderBook facade + no sys.path insert. Never bundle untrusted plugins in DMG.

### Evidence
- `loader.py:93` `exec_module`
- Zero production call sites (grep confirmed)
- `example_indicator.py:7` false auto-load claim
