### Repro
1. Start Crypcodile live connect (worker runs `run_until_complete(connector.run())`).
2. Simulate hung `transport.close()` (network stall / sleep 10s in close).
3. Call `disconnect()` → `worker.stop()` + `thread.quit()` + `wait(2000)`.
4. After 2s timeout, code sets `_worker = None` and `_thread = None` without checking `isRunning()`.
5. Thread may still run; sockets open; second `connect()` starts another worker → dual putters on shared queue.

### Expected
On wait timeout: keep ownership, surface error, block new connect; cancel tasks deterministically.

### Actual
Silent orphan after 2s; shortest timeout among providers (crypto/replay use 5s).

### Fix hint
`wait(5000+)` + if still running keep refs / refuse reconnect; cancel connector tasks + `loop.call_soon_threadsafe(loop.stop)`; never null while `isRunning()`.

### Evidence
```253:260:flowmap/data/crypcodile_live.py
def disconnect(self) -> None:
    if self._worker:
        self._worker.stop()
    if self._thread and self._thread.isRunning():
        self._thread.quit()
        self._thread.wait(2000)
    self._worker = None
    self._thread = None
```
