### Repro
1. `MainWindow.__init__` ends with `switch_to(...)` then `QTimer.singleShot(500, self._source.toggle_simulation)`.
2. `FlowmapWindow.__init__` calls `super().__init__` first (schedules live), **then** `load_historical_data` (can take long for 2h lake).
3. If hist load exceeds 500ms, live toggle may start while hist still applying snapshots (or immediately after wipe path).
4. Symbol change on SourceManager resets heatmap and restarts provider but **does not** re-invoke embed `load_historical_data` — hist is one-shot at construct.

### Expected
Documented lifecycle: hist complete → optional gap fill → then live arm. Re-hist on symbol change when embed owns data_dir. No auto-live until hist finishes (or hist_hours=0).

### Actual
Timer fires on wall clock independent of hist completion. Embed hist is construct-only; symbol switch loses preload policy.

### Fix hint
Gate auto-start until hist returns; cancel timer if load in progress; expose `reload_historical(symbol)` for symbol changes; consider `historical_hours=0` skip for pure live.

### Evidence
- `main_window.py:61-62` singleShot 500ms
- `flowmap_window.py:88-94` hist after super
- SourceManager symbol path resets without hist hook
