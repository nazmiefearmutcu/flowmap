### Repro
1. Inspect `DensityEngine.__init__`: `_buffer = np.zeros((1, 1, 4), …)` until `resize()`.
2. Inspect `FlowmapWindow.load_historical_data`: `bw = self.heatmap._engine.get_buffer().shape[1]`.
3. Trace CLI path: `run_flowmap_gui` constructs `FlowmapWindow(...)` **before** `win.show()` — hist preload runs in `__init__` while widget has never received a real geometry resize.
4. Unit check: create `HeatmapWidget()` without show/resize; assert `get_buffer().shape[1] == 1`.

### Expected
Bin count equals intended visible history width (widget pixel width / column_width, or `history_width` config), computed after layout or from an explicit target width.

### Actual
`bw` is typically **1** at preload time. Entire multi-hour window collapses into a single equal-time bin → one `push_snapshot`, catastrophic temporal compression. Interacts with gap wipe: `gap_bins >= bw` becomes `gap_bins >= 1` for any positive gap.

### Fix hint
Do not read buffer shape before show. Pass `target_bw` from screen metrics / `history_width` / post-`show()`+`resize` callback; or force `engine.resize(vis_rows, intended_bw)` before binning.

### Evidence
- `density_engine.py:77-78` — buffer starts 1×1
- `flowmap_window.py:176` — `bw = self.heatmap._engine.get_buffer().shape[1]`
- `cli.py:2549-2550` — construct then show (hist already ran)
