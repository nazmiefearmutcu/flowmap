### Repro
1. Within 50 ms of last rebuild, call `request_rebuild_throttled` → schedules `QTimer.singleShot(50, _deferred_rebuild)`, `_rebuild_pending=True`.
2. Immediate path later (`now - last > 0.05`) sets `_rebuild_pending=False` and rebuilds; or `reset()` / direct `rebuild_heatmap` without clearing pending.
3. Deferred fires: if pending cleared → **no-op** (lost trailing if a mid-window state change only set pending once); if pending still True after reset → **extra full rebuild** on new session data.

### Expected
Coalesced trailing rebuild always reflects latest geometry; cancel timer on destroy/reset; no double rebuild after mouseRelease which also checks pending.

### Actual
Flag-only cancel of logical work; singleShot always fires; mouseRelease calls `_deferred_rebuild` or full rebuild — ordering depends on flag.

### Fix hint
Use `QTimer` instance with `stop()`/`start(50)` trailing edge; clear/stop on `reset`; bind to `self` destroyed.
