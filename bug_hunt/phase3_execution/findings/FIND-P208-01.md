### Repro
1. Live path: push large wall → push empty book → rightmost column cleared to BG then redrawn (no ghost). **PASS** for `col_idx is None`.
2. Drag/history path: `_render_single_history_column` → `push_snapshot(..., col_idx=col)`.
3. Early branch:
```172:176:flowmap/engine/density_engine.py
        if col_idx is not None:
            buf_h, hw = self._buffer.shape[0], self._buffer.shape[1]
            v_rows = vis_rows if vis_rows is not None else buf_h // 5
            self._draw_column(v_rows, hw, col_idx=col_idx, update_normalizer=update_normalizer)
            return
```
4. `_draw_column` only writes `active_bids` / `active_asks` rows; **never** fills entire column with BG first.
5. If prior content had liquidity at row R and new snapshot has zero there → **stale RGB remains**.

### Expected
Any column redraw (live or historical) starts from BG then paints active levels (same contract as clear-right on live edge).

### Actual
| Path | Clear column? | Ghost risk |
|------|---------------|------------|
| Live `col_idx is None` | Yes (`buffer[:, -1] = BG`) | Low |
| Historical `col_idx=N` | **No** | High during partial redraw / drag fill |

Live scroll+clear itself is correct (S1 design intent holds for auto-follow edge only).

### Fix hint
Before `_draw_column(..., col_idx=c)`: `self._buffer[:, c, :] = ColorSystem.BG_COLOR`. Or clear inside `_draw_column` when painting a full column.
