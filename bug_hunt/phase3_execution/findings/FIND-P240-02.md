### Repro
1. Lake where historical trade AVG ≠ book_delta bid AVG (normal for multi-session or drift).
2. Start replay; read `[REPLAY_WORKER] Aligning trade prices by shift: …`.
3. Before Stage B runs, `MappedRecord` is constructed with that global `price_shift`.
4. If tracker has no BBO yet, fallback re-applies the same static shift (line 471–472).

### Expected
No silent global price offset; or window-scoped optional calibration with disclosure.

### Actual
```sql
price_shift = AVG(unnest(book_delta.bids).price) - AVG(trade.price)
```
Aggregates are **unscoped by time window** (full tables for symbol). Every trade starts with this offset; Stage B then usually overwrites, but empty-book prefixes use static shift.

### Fix hint
Delete static AVG alignment. If needed, compute only over selected `[start_ns, end_ns]` and never as silent default.
