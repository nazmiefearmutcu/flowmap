### Repro
1. Start replay with invalid symbol or window with zero book+trade rows (or wall-clock fallback range with no partitions).
2. Worker builds empty `records`, emit loop is no-op.
3. `if self._running: print(...auto-looping...);` restarts outer `while` immediately.
4. Observe tight DuckDB re-query + log spam, CPU pegged, UI lag.

### Expected
Empty result → `sig_error` / finish once; optional backoff; never tight spin.

### Actual
`while self._running:` has no sleep when `records` is empty; auto-loop is unconditional after each pass if still running.

### Fix hint
```python
if not records:
    self.sig_error.emit("No records in range")
    break
# or: awaitable sleep with exponential backoff; user toggle for loop
```
Also disable auto-loop by default; require explicit loop mode.
