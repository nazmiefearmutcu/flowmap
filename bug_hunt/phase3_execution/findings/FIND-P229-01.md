### Repro
1. Run live/replay until `_history` has hundreds of columns visible.
2. Drag-resize window wider (or taller) so `vr` or `target_bw` changes.
3. Observe left/history region of heatmap; optionally wait for many live ticks without zoom/Go Live.

### Expected
Full history re-rasterized into the new buffer dimensions (or throttled rebuild) so prior columns remain painted.

### Actual
`resizeEvent` calls `engine.resize(vr, target_bw)`, updates `_last_vis_rows` / `_last_hm_w`, then **one** `engine.push_snapshot` of current levels only. No `rebuild_heatmap()`. Next `push_snapshot` sees matching sizes → incremental path only → history stays BG/partial until zoom, Go Live, or other rebuild.

```2335:2351:flowmap/ui/heatmap_widget.py
    def resizeEvent(self, e) -> None:
        ...
        if vr != self._last_vis_rows or target_bw != self._last_hm_w:
            self._engine.resize(vr, target_bw)
            self._last_vis_rows = vr
            self._last_hm_w = target_bw
            if self._levels:
                self._engine.push_snapshot(self._levels, self._bbo, auto_follow=self.auto_follow, vis_rows=vr)
```

### Fix hint
On size change: `request_rebuild_throttled()` (or honor `engine._needs_rebuild`); do **not** stamp `_last_*` as “fully rebuilt” until rebuild completes. Regression: synthetic history + resize → non-BG pixel density in left columns.

### Evidence
- Static: resize path never calls `rebuild_heatmap`
- engine.resize only partial-copies right strip (`density_engine.py:405-421`)
