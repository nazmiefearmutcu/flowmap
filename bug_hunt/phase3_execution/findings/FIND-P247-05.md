### Repro
1. `main()` constructs `MainWindow()` with no args — always default `data_dir="/Users/nazmi/data"`.
2. No argparse/`FLOWMAP_DATA_DIR`/`FLOWMAP_HOME` in entry points.
3. SourceManager fallback list still prefers `/Users/nazmi/data` before `~/data` and `.`.
4. Crypcodile CLI **does** accept `--data-dir` for embed (`cli.py:flowmap`), but packaged FlowMap binary does not expose equivalent.

### Expected
Portable default (App Support / XDG) + `FLOWMAP_DATA_DIR` + `--data-dir` for standalone and frozen app.

### Actual
Ship binary embeds machine path (FIND-P247-01); users cannot override without editing source. Replay discovery prefers developer lake path.

### Fix hint
```python
data_dir = os.environ.get("FLOWMAP_DATA_DIR") or portable_default()
```
CLI flags on `flowmap.main:main`; remove nazmi from fallback list.

### Evidence
- `main.py` — bare `MainWindow()`
- `source_manager.py:219` — `"/Users/nazmi/data"` first in possible_dirs
