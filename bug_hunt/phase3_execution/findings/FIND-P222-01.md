### Repro
1. Live or replay running; queue has backlog (easy at speed=0 or drain starved).
2. Press Stop → `_toggle_*` sets `_running=False` **without** draining queue.
3. `_gui_tick` early-returns when `not source.running` — residual msgs sit untouched.
4. Press Start → `_running=True`; next ticks drain **old** snapshot/updates/trades into OrderBook/heatmap.
5. Contrast: `switch_to` → `stop_current` drains (good path).

### Expected
Every stop path leaves `qsize==0` or epoch-stamped drop; Start never applies pre-session messages.

### Actual
Asymmetry: only `stop_current` drains. User Stop is the common path → ghost book/history flash.

### Fix hint
Call shared `_drain_queue()` from both toggles and before `running=True` on start; optional `_gui_tick` discard-only drain when not running.
