### Repro
1. Replay any symbol with both book and trade data.
2. Capture raw lake trade prices vs emitted FlowMap `Trade.price` (via queue or signal).
3. Note log line: `Dynamic price alignment completed. Aligned N trades to order book BBO.`
4. Observe buy trades snapped to best ask, sell to best bid, else last mid.

### Expected
Emitted trade prices equal recorded lake prices (optional display-only snap if product requires it, off by default).

### Actual
After merge/sort, `LocalBookTracker` walks all records and for each `MappedRecord`:

- buy aggressor → `price_shift = best_ask - raw`
- sell aggressor → `price_shift = best_bid - raw`
- else → mid

`MappedRecord.price` returns `raw + price_shift`. `_dispatch_record` → `_cryp_trade_to_flowmap` uses `rec.price` → synthetic tape. Bubbles/VP/heatmap Y all see fake prints.

### Fix hint
Remove Stage B rewrite for production replay. If alignment remains for demo lakes, require flag + banner. Always keep `original_price` on Trade for research paths.
