### Repro
1. Construct `CryptoProvider(..., force_rest=True)` or force `ImportError` on ccxt.pro → `_start_polling`.
2. `QTimer` on provider (main thread) fires `_poll_tick`.
3. Sync `fetch_order_book` + `fetch_trades` run on caller thread for full RTT.
4. Inject network delay; observe UI freeze for duration of fetch(es).

### Expected
REST on worker QThread / asyncio; main thread only receives results.

### Actual
```python
self._poll_timer = QTimer(self)
self._poll_timer.timeout.connect(self._poll_tick)
# _poll_tick: fetch_order_book + fetch_trades synchronously
```
Queue mode does not help: puts happen **after** blocking fetches on GUI thread.

### Fix hint
Move REST to QThread or use async ccxt; keep QTimer only as scheduler that posts work. Set explicit `timeout` in ccxt config (see FIND-P223-05).
